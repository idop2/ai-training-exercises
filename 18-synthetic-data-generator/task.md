# Synthetic Data Generator: The Self-Instruct Pipeline!

## 🧠 Theory: Distilling Intelligence
Where do we get high-quality instruction data to train models like ChatGPT? Often, we use *other* models! This is known as **Synthetic Data Generation** or **Knowledge Distillation**. The "Self-Instruct" paper showed that you can start with a tiny set of human-written tasks and ask an LLM to generate *more* tasks like them, creating a massive, diverse dataset from almost nothing.

## 🚀 The Assignment
You will build a data generation pipeline that expands a small "seed" dataset into a larger training set using an LLM API.

### Steps
1.  **The Seed**: Create a `seeds.json` file with 5-10 diverse examples of tasks (e.g., "Write a poem about rust", "Explain quantum physics like I'm 5").
2.  **The Generator**: Write a script that prompts an LLM (using OpenAI/Anthropic API or a local model) to generate *new* instructions based on the seeds.
    *   *Prompt Engineering*: "Here are 3 examples. Generate 5 new, distinct tasks."
3.  **The Filter**: Implement a filter to remove low-quality or too-similar tasks (e.g., using ROUGE score or simple string overlap against the existing set).
4.  **The Solver**: For every new valid instruction, ask the LLM to generate the *response*.
5.  **The Dataset**: Save the final pairs in a JSONL format ready for fine-tuning.

## 🛠️ Tech Stack
- **Python**
- **OpenAI API** (or any LLM API/Local LLM via `ollama`)
- **Pandas** (for managing the data)

## 📦 Deliverables
1.  `generator.py`: The script that runs the generation loop.
2.  `dataset.jsonl`: A file containing at least 50 high-quality instruction-response pairs generated by your tool.
3.  `seeds.json`: Your initial manual examples.

## 🌟 Bonus Challenges
- **Diversity Check**: Use an embedding model to plot your generated tasks and visualize their diversity.
- **Topic Control**: Modify the pipeline to generate data for a *specific* domain (e.g., "Medical Advice" or "Python Coding").
